Requisitos No Funcionales

Usabilidad

Interfaz Intuitiva:

La aplicación debe tener una interfaz fácil de usar y comprender para los usuarios, independientemente de su nivel técnico. Capacitación del Usuario: Se debe proporcionar una guía o tutorial inicial para capacitar a los nuevos usuarios en el uso de la aplicación. Rendimiento

Tiempo de Respuesta:

La aplicación debe ser capaz de cargar y procesar los datos en un tiempo razonable (menos de 2 segundos por operación). Escalabilidad: La aplicación debe poder manejar un aumento en el número de usuarios y la cantidad de datos sin degradar el rendimiento. Seguridad

Autenticación y Autorización:

Debe haber un sistema seguro de autenticación y autorización para garantizar que solo los usuarios autorizados puedan acceder y modificar los datos.

Protección de Datos:

Los datos personales y de inspección deben estar protegidos mediante cifrado tanto en tránsito como en almacenamiento. Mantenibilidad

Documentación del Código:

El código de la aplicación debe estar bien documentado para facilitar el mantenimiento y las actualizaciones futuras.

Modularidad:

La arquitectura de la aplicación debe ser modular para permitir actualizaciones y mejoras sin afectar el sistema completo. Compatibilidad

Multiplataforma:

La aplicación debe ser compatible con los principales sistemas operativos móviles (iOS y Android).

Integración con Sistemas Existentes:

Debe poder integrarse con otros sistemas y bases de datos utilizados por la institución. Fiabilidad

Disponibilidad:

La aplicación debe estar disponible al menos el 99% del tiempo para garantizar que los usuarios puedan acceder a ella cuando lo necesiten.

Tolerancia a Fallos:

El sistema debe ser capaz de manejar errores y fallos sin perder datos o interrumpir significativamente el servicio. Eficiencia

Uso de Recursos:

La aplicación debe optimizar el uso de recursos del dispositivo (CPU, memoria, batería) para garantizar una experiencia fluida para el usuario. Portabilidad

Facilidad de Instalación:

La aplicación debe ser fácil de instalar y configurar en los dispositivos móviles de los usuarios.

Migración de Datos:

Debe existir un mecanismo para migrar datos desde sistemas de papel o bases de datos existentes a la nueva aplicación.
